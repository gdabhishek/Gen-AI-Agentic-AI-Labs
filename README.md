#  Generative AI and Agentic AI Labs

Hands-on labs for the **Generative AI and Agentic AI** course. A collection of practical exercises covering everything from basic prompting to advanced multi-agent systems.

## Course Context

- **Level:** Beginner to Advanced
- **Duration:** 43+ hours
- **Format:** Video lectures with hands-on coding, real-world projects, and assessments

## Topics Covered

The labs span the full course curriculum:

- **Foundation:** LLMs, LangChain, prompting, output parsing
- **RAG Systems:** Embeddings, vector databases, retrieval strategies, Corrective RAG
- **Memory & Conversation:** Chat memory, multi-turn conversations
- **Tool Calling:** Function calling, custom tools, tool chaining
- **LangGraph:** Stateful workflows, conditional routing, multi-agent coordination
- **Agentic AI:** ReAct agents, multi-agent systems, sequential workflows
- **Evaluation:** RAGAS, LangSmith monitoring, quality metrics
- **Open Source LLMs:** Hugging Face, Ollama, vLLM
- **Production:** Deployment, Streamlit apps, optimization

## Prerequisites

- Python 3.11 or higher (intermediate level)
- Basic understanding of APIs
- Virtual environment
- API keys or credentials as needed for specific labs

## Getting Started

### Setup

1. Navigate to the parent directory (one level above the lab)
2. Activate the virtual environment
3. Navigate into the lab directory
4. Install dependencies
5. Configure environment variables if required

### Running a Lab

1. Ensure the virtual environment is activated
2. Navigate to the lab directory
3. Follow the lab-specific instructions
4. Run the provided code as directed

## Learning Outcomes

By completing these labs, you will be able to:

- Build production-ready RAG applications
- Create multi-agent AI systems with LangGraph
- Implement stateful workflows and agent orchestration
- Evaluate and monitor LLM applications
- Work with both closed-source and open-source LLMs

## General Guidelines

- Activate the virtual environment before working in any lab
- Install dependencies for each lab before running code
- Store credentials in environment configuration—never in code
- API usage may incur costs depending on the service provider
